## 2. What was wrong (and what I fixed in this session)

### 2.1 CRITICAL ‚Äî fixed

| # | Issue | Status |
|---|---|---|
| C1 | `todos` table had `USING (true) WITH CHECK (true)` RLS ‚Äî anyone with the anon key (which ships in the browser bundle) could read, insert, update, or delete every row globally | **Fixed** ‚Äî table dropped in `008_lockdown_todos_and_helpers.sql`; dead `app/actions/todo-actions.ts` removed |
| C2 | `addTodo` / `toggleTodo` / `deleteTodo` server actions had no `getUser()` check ‚Äî any anonymous form post would hit the public RLS table | **Fixed** ‚Äî file deleted, actions are unreferenced |
| C3 | `next.config.ts` defined a CSP constant but never returned it from `headers()` ‚Äî production was running with **zero CSP** despite all the apparent hardening | **Fixed** ‚Äî CSP, HSTS, Permissions-Policy now actually shipped per-request |
| C4 | `eslint.ignoreDuringBuilds: true` + `typescript.ignoreBuildErrors: true` ‚Äî type errors and lint failures could ship to production silently | **Fixed** ‚Äî `ignoreBuildErrors` re-enabled; ESLint key removed (Next 16 dropped integrated ESLint, run `npm run lint` separately in CI) |
| C5 | `/api/cron/reminders` used `new Date().toISOString().slice(0,10)` (UTC) for the "is this habit done today?" check, but user-local time-of-day windowing ‚Äî a user in Asia/Kolkata at 23:30 IST has already rolled to tomorrow's calendar date, so the cron would fire reminders for already-completed habits | **Fixed** ‚Äî `currentDateInTZ(tz)` per-user, completion lookup keyed by `(user_id, local_date)` |
| C6 | Two migrations both numbered `003_*.sql` ‚Äî non-deterministic apply order | **Fixed** ‚Äî renumbered to `005_‚Ä¶`, `006_‚Ä¶`, `007_‚Ä¶` |

### 2.2 HIGH ‚Äî fixed

| # | Issue | Status |
|---|---|---|
| H1 | `/api/habits/reorder` ran N parallel `UPDATE` calls ‚Äî no transaction, partial-failure leaves the list in a half-reordered state, and any future `UNIQUE(user_id, sort_order)` constraint would deadlock | **Fixed** ‚Äî atomic `reorder_habits(p_habit_ids uuid[])` SQL RPC (SECURITY DEFINER, scoped to `auth.uid()`) |
| H2 | `HabitList` drag-end appended completed habits to the persisted order on every drag ‚Äî completed habits' `sort_order` was silently rewritten on every interaction | **Fixed** ‚Äî preserves completed habits' relative position; rolls back optimistically on save failure |
| H3 | `/api/import` skipped Zod validation ‚Äî clients could write fields they couldn't write through the regular API (e.g. spoof `current_streak`, `total_completions`) | **Fixed** ‚Äî full schema validation, 5MB body cap, schema versioning hook, structured response |
| H4 | CSV export was vulnerable to spreadsheet formula injection (cells starting with `=`, `+`, `-`, `@`) ‚Äî a malicious habit name or note becomes an executable formula when the victim opens the file in Excel/Sheets | **Fixed** ‚Äî `csvCell()` helper prefixes a single quote on injection-prefix chars, properly quotes/escapes, CRLF + UTF-8 BOM for compatibility |
| H5 | `viewport: { maximumScale: 1, userScalable: false }` ‚Äî disables pinch-zoom for low-vision users. **WCAG 1.4.4 violation**, App Store rejection risk | **Fixed** ‚Äî removed both flags |
| H6 | Brand-identity split: project named "Semma Flow", but all visible text, manifest, OG metadata, push titles, localStorage keys, custom event names, robots.txt, and sitemap said "HabitForge" | **Fixed** ‚Äî sitewide rebrand including `semma_flow_theme` localStorage key, `semma-flow:open-add` event names, push notification subject, manifest, sw.js cache name |
| H7 | `package.json` name was `habbit-tracker-claude` (typo + wrong product name) | **Fixed** ‚Äî `semma-flow` |

### 2.3 MEDIUM ‚Äî flagged, not yet applied

These are real but lower-priority than what shipped today:

- **Inline styles everywhere.** Every component has 100-300 lines of `style={{ ... }}` instead of using Tailwind v4 utilities or the design-token CSS classes already in `globals.css`. Bloats the JS bundle (every render reallocates the style object), kills Tailwind's atomic class dedupe, and makes hover/focus states verbose. Reach for `className` first.
- **God components.** `TodayHabits.tsx` is 781 lines doing onboarding, realtime, backfill modal, delete confirmation, optimistic toggle, archive, and form orchestration. Split into hooks (`useBackfill`, `useHabitMutations`) and presentational components.
- **No rate limiting** on any POST/PATCH/DELETE endpoint. Add Upstash Ratelimit or Vercel KV per-user buckets.
- **No CSRF token.** Cookie-based Supabase auth + fetch from the same origin is partially self-defending, but a CSRF token on mutating endpoints is a 30-minute add and a meaningful upgrade.
- **`/api/habits` does 3 sequential queries.** Already partially parallelized in `app/dashboard/page.tsx::fetchOverviewStats` ‚Äî same pattern needs to land in the route handlers used by the client palette and detail pages.
- **`useMotionValue` animation in `OverviewStats`** ‚Äî every count animation creates a fresh subscription and does manual `textContent` writes. Works, but bypass React reconciliation; if you ever wrap this in a list, watch GC churn.
- **Service worker has no version bust on deploy.** `CACHE_NAME = 'semma-flow-v1'` is static ‚Äî bumping requires a code change. Hash from build time instead.
- **`IconPicker` ships every Lucide icon name as runtime PascalCase lookup** in `HabitCard::DynamicIcon`. Tree-shaking can't help ‚Äî ship a curated subset (already declared in `lib/constants.ts::HABIT_ICONS`).
- **Onboarding flag stored in `localStorage` only.** Cross-device reset gives the wizard again. Mirror to `profiles.onboarded_at`.
- **`OAuth callback "isNewUser" detection via 10-second timestamp diff`** is fragile ‚Äî a slow Google round-trip exceeds the threshold. Use `auth.users` row count or a profile `created_at` check instead.
- **`safeErrorMessage` swallows context in prod.** Log a request-scoped error ID + return that ID to the user; engineers can `grep` logs.
- **`reminder_time` stored as `TIME` (no TZ).** Today the cron coerces by user TZ, but if a user changes timezone, all their existing reminders silently shift. Persist the time + the TZ it was set in.
- **`isHabitDueToday` uses server `getDay()`** ‚Äî same TZ trap as the cron used to have. Pipe user TZ through.

## 5. Standout-globally features

A habit tracker is a saturated category. To stand out:

- **Causal correlation surfacing** ‚Äî "On days you slept 7+ hours you completed 84% more habits." This is novel; nobody ships it well. Requires logging sleep separately (HealthKit) or asking user.
- **Voice logging** ‚Äî "Hey Semma, mark all morning habits done" via Web Speech API + service worker.
- **Embed API** ‚Äî single-habit streak SVG that any blogger / Notion user can embed; viral acquisition channel.
- **Carbon-neutral streaks** ‚Äî partner with a tree-planting API; for every 30-day streak, plant a tree. Brand-aligned ("growth").
- **End-to-end encrypted notes** ‚Äî habit notes encrypted client-side; differentiator vs. every privacy-careless competitor.

Pick **one** of these to anchor a launch narrative ‚Äî don't ship all five.
